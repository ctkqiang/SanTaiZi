# 如何提交模块

## 1. 模块开发流程

### 1.1 环境准备
1. 安装Go开发环境（版本1.18+）
2. 克隆三太子框架代码库
3. 熟悉框架结构和模块加载机制

### 1.2 模块开发
1. 在`modules`目录下创建合适的子目录，如`scanners`、`http`等
2. 创建模块源文件，遵循命名规范
3. 实现模块基本功能，包括：
   - 模块信息输出（--info）
   - 模块执行逻辑（--execute）
   - 选项定义和处理

### 1.3 模块测试
1. 编译模块：`go build -o modules/bin/模块名称 模块源文件`
2. 测试模块信息输出：`./modules/bin/模块名称 --info`
3. 测试模块执行：`./modules/bin/模块名称 --execute 参数1=值1 参数2=值2`
4. 测试模块在框架中的集成：`./modules/bin/santaizi`

## 2. 模块提交规范

### 2.1 提交前检查
1. 确保模块代码符合代码标准
2. 确保模块通过所有测试
3. 确保模块安全，无安全漏洞
4. 确保模块文档完整

### 2.2 提交内容
1. 模块源文件
2. 模块说明文档（如有）
3. 模块测试文件（如有）
4. 模块依赖说明（如有）

### 2.3 提交流程
1.  Fork代码库到个人账号
2.  创建新分支，命名为`feature/模块名称`
3.  提交代码到分支
4.  提交Pull Request，包含以下信息：
   - 模块名称和功能描述
   - 模块实现细节
   - 测试结果
   - 依赖说明
   - 安全评估

## 3. 模块审核标准

### 3.1 功能审核
- 模块功能是否完整
- 模块是否满足需求
- 模块是否与框架兼容

### 3.2 代码审核
- 代码是否符合代码标准
- 代码是否安全，无安全漏洞
- 代码是否高效，无性能问题
- 代码是否可维护，有适当的注释

### 3.3 测试审核
- 模块是否通过所有测试
- 测试覆盖率是否达标
- 测试是否充分，覆盖边界情况

### 3.4 文档审核
- 模块是否有完整的说明文档
- 文档是否清晰明了
- 文档是否包含使用示例

## 4. 模块发布

### 4.1 发布流程
1. 模块通过审核后，合并到主分支
2. 编译模块并发布到`modules/bin`目录
3. 更新模块列表和版本信息

### 4.2 版本管理
- 模块应使用语义化版本规范
- 版本变更应记录在CHANGELOG中
- 重大变更应在版本更新中说明

### 4.3 模块维护
- 模块开发者应定期维护模块
- 及时修复模块中的bug和安全漏洞
- 根据框架更新调整模块代码

## 5. 模块使用指南

### 5.1 模块安装
1. 下载模块到`modules`目录
2. 编译模块：`go build -o modules/bin/模块名称 模块源文件`
3. 启动框架：`./modules/bin/santaizi`
4. 搜索并使用模块：`search 模块名称`

### 5.2 模块配置
- 使用`set`命令设置模块选项
- 使用`options`命令查看模块选项
- 配置完成后使用`run`命令执行模块

### 5.3 模块输出
- 模块执行结果会显示在控制台
- 错误信息会包含`error:`前缀
- 详细输出会根据模块实现有所不同

## 6. 常见问题

### 6.1 模块加载失败
- 检查模块是否编译成功
- 检查模块是否有执行权限
- 检查模块信息输出是否符合要求

### 6.2 模块执行失败
- 检查模块选项是否正确设置
- 检查模块依赖是否安装
- 检查模块代码是否有错误

### 6.3 模块性能问题
- 检查模块代码是否有性能瓶颈
- 检查模块资源使用是否合理
- 考虑优化模块算法和数据结构

### 6.4 模块安全问题
- 检查模块是否存在安全漏洞
- 遵循安全规范进行修复
- 及时更新模块以修复已知安全问题
